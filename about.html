About this website

After completing my course project on social media misinformation, I wanted to publish an article online detailing my experiences and thoughts from its inception to completion. I realized that I might want to showcase other personal projects I’ve uploaded to Github with their own articles. While I could have used Medium or another social media site to do so, it only seemed natural to use a personal website which itself can also be managed with Github.

There are a few caveats to using Github Pages to host a website. Among them are a 1GB limit on the repository from which the website is published, and that the website cannot be used for commercial purposes. Otherwise, it’s useful for showcasing several select things, like this website does.

I’ve written instructions for readers to make their own websites like this one if they’re interested. Once you’ve learned a little HTML/CSS, it doesn’t take long to get your own website up and running for free.

How to make your own personal website:

For accessibility, I have written with the assumption that the reader has little or no knowledge of web development.

Before starting, you should have the following:

  • A Github account. Github Pages will allow you to publish and maintain a static website hosted in a Github repository for free.
  • A Formspree account. Formspree makes it simple to set up and receive responses from forms on your website without requiring knowledge of backend (server-side) web development. If you don’t plan on having forms on your website, this is unnecessary.
  • Some basic understanding of HTML/CSS, to customize a template (if you choose to use one) or create the necessary files for the website from scratch.

To start, you can download a free website template from sites such as HTML5 UP, and modify it to suit your preferences. If you can’t find a template that you like, feel free to code your own HTML and CSS files. No matter what design you settle on, make sure that it fits the content you plan to publish, is simple (so that your website will load faster for impatient viewers), and accessible to visitors using devices with varying screen sizes (e.g. the content will load or fit on not only large computer screens). Downloading the right template can take care of these problems for you. I personally downloaded the Phantom template from HTML5 UP. Note that you can see how your website will look before publishing it by opening the .html file with an Internet browser like Google Chrome.

If your website design includes a form, you can connect it to your Formspree account so that responses will be delivered to your email inbox. Once you’ve created a new project and form on your Formspree account, paste the form URL into the “action” attribute of your opening form tag. Make sure that the “method” attribute is set to “post”.

Once you have finished coding your website, you can create and populate the Github repository from which your website will be hosted. Depending on what you name the repository, it will either be a user site or a project site, and the URL will change accordingly.

User site Project site
Repository name [username].github.io [repository_name]
URL [username].github.io [username].github.io/[repository_name]
Quantity allowed 1 per Github account Unlimited

If you want your website to have a custom URL, you can purchase one and specify it in a CNAME file in your repository.

You can then upload the files to the repository using Github’s GUI or a command-line interface (CLI). If you choose to use the CLI, use the following commands (note that if you are using Windows, you will need to download a special terminal to use Git):

  1. In the directory containing your repository, use git init to initialize Git.
  2. Connect to Github using git remote add origin https://github.com/[URL].git, where [URL] is your website's URL (see the above table).
  3. Select all of the files in your repository with git add .
  4. Commit your files with git commit -m “[message]”, where you can briefly summarize the changes in the commit in [message].
  5. Finally, push your changes with git push origin main. If your main branch is named something other than "main", specify that name instead of main at the end of that command.

To publish new changes to your website, repeat the above steps except the 2nd. Note that it may take some time for Github Pages to register your commit and update your website, especially when you are first creating your website.

If you find any of the steps above unclear, you can refer to this tutorial. You can also see the repository for this website here.
